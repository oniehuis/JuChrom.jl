Units and Quantities

JuChrom supports optional units via Unitful.jl. You can work with unitless values or Unitful quantities interchangeably, as long as each operation uses compatible units. This page gives a short, package‚Äëfocused primer on the syntax you need for JuChrom.

Quick primer

  • Attach a unit to a numeric value with the u"" string macro:
using JuChrom  # reexports Unitful

rt = 1.0u"s"  # (seconds)
1.0 s
rt = 12.5u"minute"
12.5 minute
mz = 100.0u"Th"  # (Thomson)
100.0 Th
signal = 5.3u"pA"  # (picoampere)
5.3 pA
  • Attach units to arrays in the same way:
rts = [1.0, 2.0, 3.0]u"s"
3-element Vector{Quantity{Float64, ùêì, Unitful.FreeUnits{(s,), ùêì, nothing}}}:
 1.0 s
 2.0 s
 3.0 s
  • Convert between compatible units with uconvert:
rt = uconvert(u"minute", 120.0u"s")
2.0 minute
  • Convert units for every element of a vector with broadcasting:
rts = uconvert.(u"minute", [30.0, 60.0, 90.0]u"s")
3-element Vector{Quantity{Float64, ùêì, Unitful.FreeUnits{(minute,), ùêì, nothing}}}:
 0.5 minute
 1.0 minute
 1.5 minute
  • Remove units when you need raw numbers:
rawrt = ustrip(12.5u"s")
12.5
  • Remove units from every element of a vector with broadcasting:
rawrts = ustrip.([30.0, 60.0, 90.0]u"s")
3-element Vector{Float64}:
 30.0
 60.0
 90.0
  • Unitful checks dimensional consistency for you:
rt = 1.0u"s" + 500u"ms"  # is valid
1.5 s
try
    rtmz = 1.0u"s" + 1.0u"Th"  # raises an error
catch err
    println(err)
end
Unitful.DimensionError(1.0 s, 1.0 Th)

JuChrom accepts either unitless or unitful inputs for retentions, intensities, and m/z values. If you pass unitful data, JuChrom preserves and propagates units where appropriate. If you pass unitless data, the corresponding unit is stored as nothing.

Common units in JuChrom

Below are the units you will see most often in this package:

  • Time: u"s" (second), u"ms" (millisecond), u"minute"
  • Current (intensity units for some detectors, e.g., FID): u"A" (ampere), u"mA", u"ŒºA", u"nA", u"pA"
  • Mass‚Äëto‚Äëcharge: u"Th" (Thomson; optional), or unitless m/z

Metric prefixes

Unitful follows the International System of Units (SI) prefixes. The most common are:

  • milli (m): 1e-3 (e.g., u"ms", u"mA")
  • micro (Œº): 1e-6 (e.g., u"Œºs", u"ŒºA")
  • nano (n): 1e-9 (e.g., u"ns", u"nA")
  • pico (p): 1e-12 (e.g., u"ps", u"pA")
